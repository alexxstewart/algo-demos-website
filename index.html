How It Works

The Interactive Demonstration above showcases a simple way of navigating a map from on point to another. There are many path finding algorithms that can be used, all with different methods and use cases. Some of these are the A* algorithm, the sample algorithm, and the algorithm used in this demonstration: Djikstra's algorithm.

Djikstra's Algorithm

This algorithm comes in many different variations for different use cases but here we will just be explaining the more common approach at finding the shortest path between a start node and end node.

This algorithm falls under the branch of algorithms called greedy algorithms. This is becausde for each node that the algorithm processes it select the locally optimal node (shortest distance) to travel to next.

There are 6 main steps to Djikstra's Algorithm:

  1. Mark all nodes as unvisited. We can do this by placing them in a separate set
  2. Assign all nodes a temporary distance value. For unvisited nodes we set this to Infinity, and the starting node is set to 0
  3. For the current node, consider all of its unvisited neighbours and calculate their distances through the current node. Compare the newly calculated distance to the current assigned value and assign the smaller one. For example, if the current node A is marked with a distance of 6, and the edge connecting it with a neighbour B has length 2, then the distance to B through A will be 6 + 2 = 8. If B was previously marked with a distance greater than 8 then change it to 8. Otherwise, the current value will be kept.
  4. When we are done checking all univisted neighbours of the current node, we mark the node as visited and remove it from the unvisited set. A visited node will never be checked again
  5. If the destination node has been marked as visited or if the smallest path to this node is Infinity (no path to this node), then we terminate the algorithm
  6. Otherwise, we select the univisted node that has the smallest distance value from the current node and set it to the current node.

Applications

While the original variation of Djikstra's algorithm (used above) has few applications in the real world, variations are built of the idea which can be better suited and further optimised for a specific task. Some example of real world uses are:

  • Some variations are used in computing shortest paths in computer networks to determine ways to make data transfer easier and more effective.
  • Some heavily modified versions are used in computing shortest paths between two locations on a map.
  • They are used to help establiush tracks of electricity lines and oil pipelines.
  • The algorithm has also been used to calculate optimal long-distance footpaths in Ethiopia and contrast them with the situation on the ground
